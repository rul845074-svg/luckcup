# LuckCup 数据分析与规范化录入报告

## 一、数据来源

所有历史数据来自 Apple Numbers 格式的月度账单（`毛庄清单/核对清单*.numbers`），涵盖 2024 年底至 2026 年初，共 20+ 个文件。每个文件按月划分工作表，每月有"收入"和"支出"两张表。

补充数据源：
- `毛庄报货2025.xlsx` / `毛庄报货2026.xlsx` — 每次报货的具体货品明细

---

## 二、数据现状问题

### 类别碎片化严重
- 收入类别 68 个，支出类别 159 个
- 同一业务有多种写法（如"报货"、"冷链"、"普货冷链"实质都是进货）
- 部分类别混合了多种业务（如"佳一工资吸管"、"咖啡豆纸袋拿铁"）

### 前台日销和非经营收支混在一起
- 门店日销（雪王、美团外卖等）和非标收入（调货、废品、转卖原料）没有区分
- 导致月度总览的"总收入"包含了不属于日常经营的金额，盈亏分析失准

---

## 三、数据整理逻辑

### 核心原则：前台日录 vs 后台补录

将所有收支分为两类：

**前台日录** — 老板每天录入的标准经营数据
- 收入 6 个渠道：美团团购、美团外卖、外卖其他、抖音团购、小程序、门店收银
- 支出 6 个类别：原料货品、周边货物、工资、房租、水电、其他支出

**后台补录** — 不该让老板操心、但需要记录的非标数据
- 收入：调货收入、B端/客户收入、商品转卖收入、营业外收入、废品/退款等
- 支出：分红/老板支出、税费/管理费

### 收入映射规则

| 原始类别 | 归入渠道 | 逻辑 |
|---------|---------|------|
| 雪王、现金、云闪付 | 门店收银 | 门店当面交易统一口径 |
| 美团外卖、美团 | 美团外卖 | 保持原口径 |
| 饿了么、京东 | 外卖其他 | 非美团的到家外卖统一 |
| 抖音、快手 | 抖音团购 | 短视频平台统一 |
| 在线点、美团在线点 | 小程序 | 线上自助点单统一 |
| 冰淇淋、脆筒、柠檬等食材名 | 后台-商品转卖收入 | 不是日销渠道，是原料转卖 |
| 调货、借货、帮报货 | 后台-调货收入 | 门店间往来，不算日销 |
| 利息、废品、退款 | 后台-营业外收入 | 非经营收入 |

### 支出映射规则

| 原始类别 | 归入类别 | 逻辑 |
|---------|---------|------|
| 报货、冷链、各种原料名 | 原料货品 | 所有进货统一（原名"普货"） |
| 杯子、吸管、纸袋等包材 | 原料货品 | 包材耗材也并入原料货品 |
| 欧包、拿铁条、面包干等 | 周边货物 | 独立售卖的周边商品 |
| 各种人名+工资、预支 | 工资 | 统一口径 |
| 房租、房租（半） | 房租 | 统一 |
| 电费、水费、宽带 | 水电 | 固定费用统一 |
| 维修、设备、活动物料等 | 其他支出 | 原名"突发支出"，改为更准确的命名 |
| 分红 | 后台-分红/老板支出 | 不放前台 |
| 增值税、管理费 | 后台-税费/管理费 | 不放前台 |

## 四、数据在系统中的流向

```
Numbers 历史账单
    │
    ├─ 前台日录数据 ──→ daily_income 表（收入）/ expenses 表（支出）
    │                     │
    │                     ↓
    │               月度总览（总收支、平台/类别占比）
    │               盈亏分析（利润率、各平台贡献）
    │               盈亏平衡点（固定成本+采购+人力+水电）
    │
    └─ 后台补录数据 ──→ backend_entries 表
                          │
                          ↓
                    分析报表中单独列示，计入总收支但不混入日销渠道
```

## 五、后续规范化录入指南

### 老板日常只需做两件事

1. **每天录一次收入**：6 个渠道各填当日到账金额
2. **发生支出就记一笔**：选类别、填金额、写备注

### 备注规范

| 场景 | 备注写法 |
|------|---------|
| 冷链单独付款 | 原料货品，备注"冷链-货名" |
| 报货充值/预存 | 原料货品，备注"预存"或"充值" |
| 帮其他店报货 | 原料货品，备注"帮报-店名" |
| 预支工资 | 工资，备注"预支-人名" |
| 混合付款（如工资+吸管） | 按金额大头分类，备注说明另一项 |

### 后台补录由谁做

调货、转卖、废品回收、分红等非日常项目，在"后台补录"页面单独记录，不会出现在老板的日常录入界面中，但会计入月度分析。
