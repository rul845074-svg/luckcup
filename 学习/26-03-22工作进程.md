## 今日 DNS / COS 排障记录

### 问题

访问 `lcup.cn` 返回 COS 的 XML 错误 `AccessDenied`。

### 根因

COS 有两个端点：
- **普通端点** `xxx.cos.ap-shanghai.myqcloud.com` — 当作对象存储用，访问 `/` 没有对应对象 → AccessDenied
- **静态网站端点** `xxx.cos-website.ap-shanghai.myqcloud.com` — 当作网站用，访问 `/` 自动返回 `index.html`

DNS 之前指向了普通端点，所以根路径打不开。

### 历史矛盾

- DNS 指向 `-website` 端点 → 浏览器能访问，但 COS 控制台报 CNAME 冲突
- DNS 指向普通端点 → COS 控制台不报错，但浏览器根路径 AccessDenied

### 解决方案

1. DNSPod 的 CNAME 指向 `-website` 端点
2. COS 控制台自定义域名的**源站类型**从"默认源站"改为"静态网站源站"
3. 两端必须匹配，否则 COS 返回 403

此问题由 Kimi 联网查到腾讯云文档后定位。Claude 能排查到 DNS 端点问题，但不知道 COS 控制台有"源站类型"这个配置项。
